# Terraform Learning - Day 1

Today I started my Terraform journey and learned the basics of Infrastructure as Code (IaC) using AWS.

## What I Learned

* Installed Terraform on Linux.
* Learned about HashiCorp and the purpose of Terraform.
* Explored Terraform documentation and understood how Terraform works.
* Learned about Terraform Providers and how they help Terraform interact with cloud platforms like AWS.
* Configured AWS credentials to allow Terraform to manage AWS resources.
* Understood the Terraform workflow and commonly used commands:

  * `terraform init`
  * `terraform plan`
  * `terraform apply`
* Got familiar with HashiCorp Configuration Language (HCL).

## Hands-On Practice

As part of my first Terraform exercise, I created an AWS VPC using Terraform instead of creating it manually through the AWS Console.

This helped me understand:

* Provider configuration
* Resource blocks
* Infrastructure provisioning using code

## Key Takeaways

* Terraform allows infrastructure to be managed in a consistent and repeatable way.
* Providers act as a bridge between Terraform and cloud services.
* The typical Terraform workflow is:

  1. Write the configuration
  2. Initialize the project
  3. Review the execution plan
  4. Apply the changes
